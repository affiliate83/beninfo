# 제품 요구사항 정의서 (PRD): 정부지원금 및 채용정보 맞춤 추천 포털 (워드프레스 기반)

## 1. 개요 (Overview)
- **프로젝트명:** BenefitInfo (가칭)
- **목적:** 대한민국 정부 지원정책, 복지 혜택, 공공기관 채용 공고를 한 곳에서 쉽게 검색하고 사용자 맞춤형으로 추천받을 수 있는 웹사이트를 구축합니다. (벤치마킹 대상: [국민자료실 - govhelp.co.kr](https://govhelp.co.kr/))
- **타겟 고객:** 정부 지원 혜택 및 공공기관 채용 정보를 찾고자 하는 청년, 구직자, 소상공인, 그리고 일반 국민
- **개발 환경:** WordPress (워드프레스 CMS)

---

## 2. 벤치마킹 분석 (govhelp.co.kr 핵심 요소)
- **직관적인 카테고리:** 청년, 복지, 주거, 고용, 창업, 교육, 보건의료, 문화생활, 농림어업 등 사용자 관심사 중심의 분류
- **공고 마감일 강조:** 'D-Day' 표시를 통해 사용자에게 시급성을 알리고 액션 유도
- **빠른 검색 및 필터링:** 인기 검색어(키워드) 퀵 링크 및 통합 검색 기능
- **데이터 신뢰성:** 공공데이터포털 등 공식 기관의 데이터를 바탕으로 정보 제공

---

## 3. 핵심 기능 요구사항 (Core Features)

### 3.1 사용자 인터페이스 (UI/UX)
- **메인 페이지 (Home):**
  - 통합 검색바 (키워드 검색)
  - 주요 카테고리 아이콘 네비게이션 (청년, 복지, 주거, 고용, 창업, 교육, 보건의료, 농림어업)
  - 마감 임박 (D-Day) 공고 리스트 슬라이더
  - 신규 등록된 지원사업 / 채용 공고 리스트
- **카테고리 및 목록 페이지 (Archive):**
  - 카테고리별 공고 목록 출력
  - 정렬 기능 (마감일순, 최신순, 조회수순)
  - 페이지네이션 및 무한 스크롤
  - 마감 임박 배지 (D-3, D-7 등) 표시
- **상세 페이지 (Single):**
  - 공고 상세 내용 (지원 대상, 지원 내용, 신청 방법, 신청 기간 등)
  - 공식 신청 페이지로 이동하는 '외부 링크 버튼 (Call To Action)'
  - 연관 공고 추천 (관련 카테고리 내 다른 글)
  - 소셜 공유 버튼 (카카오톡, 카카오스토리, 링크 복사)
- **모바일 최적화:**
  - 반응형(Responsive) 디자인 필수
  - 모바일 기준 핵심 정보(D-Day, 지원 대상, 신청 링크) 상단 노출
  - 터치 친화적인 버튼 크기 및 간격 (최소 44px)

### 3.2 검색 및 필터
- 키워드 검색 (제목 및 내용 기반)
- 태그(Tag) 시스템을 활용한 다중 필터링 (예: #청년, #채용, #지원금 등)
- 지역 필터 (전국 / 서울 / 경기 / 광역시 등)
- 대상 필터 (청년 / 중장년 / 장애인 / 소상공인 / 농어업인 등)
- 마감 여부 필터 (진행 중 / 마감 포함)

### 3.3 알림 및 구독 기능 (선택 구현)
- 이메일 뉴스레터 구독 (관심 카테고리 기반 주간 요약)
- 카카오 채널 또는 텔레그램 봇 연동 (마감 임박 공고 알림)

## 4. 워드프레스 시스템 아키텍처 (WordPress Architecture)

### 4.1 데이터 구조 설계 (Custom Post Type & Taxonomy)
- **Custom Post Type (CPT):** `policy` (정부지원정책)
- **Custom Taxonomy (분류):**
  - `policy_category`: 청년, 복지, 주거, 고용, 창업, 교육, 보건의료, 문화생활, 농림어업
  - `policy_region`: 전국, 서울, 부산, 대구, 인천, 광주, 대전, 울산, 세종, 경기, 강원, 충북, 충남, 전북, 전남, 경북, 경남, 제주
  - `policy_target`: 청년, 중장년, 신혼부부, 장애인, 다문화가정, 소상공인, 농어업인, 취업준비생
- **Custom Fields (ACF - Advanced Custom Fields 활용):**
  - `application_start_date` (신청 시작일)
  - `application_end_date` (신청 마감일) → D-Day 계산에 사용
  - `target_audience` (지원 대상 요약)
  - `benefit_summary` (지원 내용 요약)
  - `official_link` (공식 신청/공고 링크)
  - `agency_name` (주관 기관명)
  - `support_scale` (지원 규모, 예: 최대 300만원)
  - `data_source` (데이터 출처 API명 또는 URL) → 중복 방지 및 추적용
  - `is_expired` (마감 여부 Boolean) → 검색 필터링용

### 4.2 추천 테마 및 플러그인 구성
- **테마:** GeneratePress 또는 Astra (가볍고 빠른 로딩 속도, SEO 최적화에 유리)
- **필수 플러그인:**
  - **Advanced Custom Fields (ACF):** 상세 정보 메타필드 관리
  - **Custom Post Type UI (CPT UI):** 포스트 타입 및 텍소노미 생성
  - **WP All Import:** 외부 데이터(공공데이터포털 XML/JSON) 대량 크롤링 및 워드프레스 자동 포스팅 연동
  - **Rank Math SEO / Yoast SEO:** 검색엔진 최적화
  - **WP Super Cache / LiteSpeed Cache:** 페이지 로딩 속도 개선
  - **Wordfence Security:** 워드프레스 보안 방화벽 및 악성코드 탐지
  - **UpdraftPlus:** 자동 백업 (일별 또는 주별)
  - **Simple 301 Redirects:** URL 변경 시 리디렉션 관리

## 5. 데이터 수집 및 자동화 (Data Pipeline)

### 5.1 데이터 소스
- **공공데이터포털 (data.go.kr):** 복지로, 보조금24, 청년정책 API
- **워크넷 API (work.go.kr):** 공공기관 채용 공고
- **복지로 (bokjiro.go.kr):** 사회복지 서비스 정보
- **K-Startup:** 창업 지원 사업 정보

### 5.2 자동화 로직
1. 외부 Python 크롤러 또는 워드프레스 Cron Job을 이용하여 주기적(일 1회 등)으로 API 데이터 수집
2. 수집된 JSON/XML 데이터를 WP All Import를 통해 워드프레스 `policy` CPT로 자동 업데이트 (신규 생성 및 마감된 공고 상태 변경)
3. 만료된 공고(신청 기한 초과)는 자동으로 `is_expired = true` 설정 및 목록에서 제외 처리
4. `data_source` 필드 기반 중복 방지 (동일 공고 재업로드 차단)

### 5.3 콘텐츠 품질 관리
- 수집된 원본 데이터에 **요약 정보 자동 생성** 적용 (Claude API 또는 GPT API 활용 검토)
- 제목이 없거나 내용이 지나치게 짧은 항목은 자동 임시저장(Draft) 처리 후 수동 검토
- 월 1회 주기로 만료 공고 일괄 정리 및 DB 최적화

### 5.4 에러 처리 및 모니터링
- API 호출 실패 시 최대 3회 재시도 후 슬랙(Slack) 또는 이메일 알림 전송
- 크롤러 실행 로그 파일 보관 (최근 30일)
- 일일 데이터 수집 건수 및 오류 건수 대시보드 (Google Sheets 또는 Notion 활용)

## 6. SEO 전략 (SEO Strategy)

### 6.1 기술적 SEO
- 각 공고별 고유 메타 타이틀/디스크립션 자동 생성 (공고명 + 주관기관 + 신청기간 조합)
- Open Graph 및 Twitter Card 태그 적용 (소셜 공유 시 썸네일 표시)
- XML 사이트맵 자동 생성 및 Google Search Console 등록
- 구조화 데이터(Schema Markup) 적용 - `GovernmentService` 또는 `Event` 타입

### 6.2 콘텐츠 SEO
- 카테고리별 허브 페이지에 상위 노출을 위한 소개 텍스트 작성 (예: "2026년 청년 지원정책 총정리")
- 롱테일 키워드 타겟 (예: "만 19세 청년 월세 지원금 신청 방법")
- 인기 공고는 자동으로 메인 페이지에 노출되도록 조회수 기반 정렬 적용

## 7. 보안 및 법적 고려사항 (Security & Legal)

### 7.1 보안
- 워드프레스 관리자 계정 이중 인증(2FA) 적용
- REST API 엔드포인트 불필요한 노출 최소화 (`/wp-json` 인증 없이 접근 제한)
- SSL 인증서 적용 (HTTPS 필수)
- 정기 보안 업데이트 (워드프레스 코어, 테마, 플러그인)

### 7.2 법적 고려사항
- **공공데이터 활용 라이선스:** 공공데이터포털 API 이용약관 준수, 원문 출처 명시 의무
- **개인정보 처리방침:** 이메일 뉴스레터 구독 시 개인정보 수집 동의 및 처리방침 페이지 필수
- **저작권:** 수집된 이미지/콘텐츠의 저작권 확인 후 사용, 원본 출처 표기
- **광고 표시 의무:** 제휴 마케팅 링크에 '광고' 또는 '제휴' 표기 (공정거래위원회 고시 준수)

---

## 8. 수익화 모델 (Monetization Strategy)
- **디스플레이 광고:** 구글 애드센스(Google AdSense) 연동을 통한 배너 수익 (메인 페이지, 목록 사이, 상세 페이지 본문 상/하단 배치)
- **제휴 마케팅 (Affiliate):** 정부 지원금 신청에 필요한 인증서 발급 안내, 관련 금융 상품(신용대출, 청년도약계좌 등) 제휴 링크 삽입 (쿠팡 파트너스, 텐핑 등)
- **애드센스 승인 전략:** 최소 포스트 50개 이상 + 카테고리별 허브 페이지 완성 후 신청

## 10. 마일스톤 (Milestone)
1. **1단계 (기획 및 세팅):** 워드프레스 설치, 테마/플러그인 세팅, CPT 및 ACF 필드 구조 설계
2. **2단계 (데이터 연동):** 공공데이터 API 분석 및 WP All Import 크롤링 파이프라인 구축 테스트
3. **3단계 (UI/UX 개발):** 메인페이지, 목록페이지(D-Day 표시 등), 상세페이지 템플릿 퍼블리싱 및 적용, 모바일 반응형 검수
4. **4단계 (SEO 기반 콘텐츠):** 카테고리 허브 페이지 작성, 구조화 데이터 적용, 구글 서치 콘솔 등록
5. **5단계 (오픈 및 최적화):** 테스트 데이터 검증, 애드센스 승인 요청, 모니터링 대시보드 설정 및 정식 오픈
